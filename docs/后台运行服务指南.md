# 后台运行服务指南

## 📋 概述

使用 Docker Compose 可以让前后端服务在后台持续运行，即使关闭终端也不会中断。

## 🚀 快速开始

### 1. 启动所有服务（后台运行）

```bash
make docker-up
```

**说明**：
- `-d` 参数表示后台运行（detached mode）
- 服务会在后台持续运行，不依赖终端窗口
- 启动后可以安全关闭终端

### 2. 查看服务状态

```bash
make docker-status
# 或直接使用
docker compose ps
```

### 3. 查看实时日志

```bash
make docker-logs
# 或直接使用
docker compose logs -f
```

**查看特定服务日志**：
```bash
# 只看后端日志
docker compose logs -f backend

# 只看前端日志
docker compose logs -f frontend

# 只看数据库日志
docker compose logs -f mysql
```

### 4. 重启服务

```bash
make docker-restart
# 或单独重启某个服务
docker compose restart backend
```

### 5. 停止所有服务

```bash
make docker-down
```

## 🔧 常用命令

| 命令 | 说明 |
|------|------|
| `make docker-up` | 后台启动所有服务 |
| `make docker-down` | 停止所有服务 |
| `make docker-logs` | 查看所有日志 |
| `make docker-restart` | 重启所有服务 |
| `make docker-status` | 查看服务状态 |
| `docker compose logs -f [服务名]` | 查看特定服务日志 |
| `docker compose stop [服务名]` | 停止特定服务 |
| `docker compose start [服务名]` | 启动特定服务 |

## 🌐 访问服务

启动后，可以通过以下地址访问：

- **前端服务**: http://localhost:5173
- **后端 API**: https://localhost:8000
- **API 文档**: https://localhost:8000/docs
- **MySQL 数据库**: localhost:3306

## 💡 使用场景

### 场景 1：开发时保持服务运行

```bash
# 早上启动服务
make docker-up

# 开始一天的工作...
# 中间可以随意关闭终端、重启电脑

# 需要查看日志时
make docker-logs

# 下班时停止服务
make docker-down
```

### 场景 2：只重启后端服务

```bash
# 修改了后端代码后
docker compose restart backend
```

### 场景 3：监控服务状态

```bash
# 定期检查服务运行状态
make docker-status
```

## ⚙️ Docker Compose 配置说明

项目使用 `docker-compose.yml` 定义三个服务：

1. **mysql**: 数据库服务
2. **backend**: Python FastAPI 后端
3. **frontend**: Vue.js + Vite 前端

所有服务都配置了：
- 自动重启策略
- 数据持久化卷
- 网络隔离

## 🔍 故障排查

### 服务启动失败

```bash
# 查看详细日志
docker compose logs

# 检查端口占用
lsof -i:8000
lsof -i:5173
lsof -i:3306
```

### 容器意外退出

```bash
# 查看退出的容器
docker compose ps -a

# 查看错误日志
docker compose logs backend
```

### 清理并重建

```bash
# 完全清理（会删除数据卷）
docker compose down -v

# 重新构建并启动
docker compose up -d --build
```

## 📝 注意事项

1. **数据持久化**: MySQL 数据存储在 Docker 卷中，即使容器删除也不会丢失
2. **资源占用**: 后台运行的服务会持续占用系统资源
3. **开机自启**: 如需开机自动启动，可配置 systemd 服务
4. **日志管理**: 定期清理日志避免占用过多磁盘空间

## 🎯 最佳实践

1. **开发环境**: 使用 `make docker-up` 启动，需要时再用 `make docker-logs` 查看日志
2. **调试问题**: 使用 `docker compose logs -f` 实时跟踪日志
3. **代码更新**: 后端支持热重载，前端 Vite 也会自动刷新
4. **定期清理**: 使用 `docker system prune` 清理未使用的资源
