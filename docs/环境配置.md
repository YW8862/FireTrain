## 1. 环境要求

### 1.1 基础软件

| 软件 | 版本要求 | 用途 |
|------|---------|------|
| Python | 3.9+ | 后端开发 |
| Node.js | 18+ (推荐 20+) | 前端开发 |
| npm | 10+ | 前端包管理 |
| Docker | 24+ | 容器化部署 |
| Docker Compose | V2+ | 服务编排 |

### 1.2 系统要求

- **操作系统**: Windows/Linux/MacOS
- **内存**: 最低 4GB，推荐 8GB+
- **存储**: 至少 10GB 可用空间

## 2. 本地开发环境配置

### 2.1 后端环境配置

#### Step 1: 创建虚拟环境（可选但推荐）

```bash
cd backend
python -m venv .venv

# Windows
.venv\Scripts\activate

# Linux/Mac
source .venv/bin/activate
```

#### Step 2: 安装依赖

```bash
pip install -r requirements.txt
```

#### Step 3: 配置环境变量

复制 `.env.example` 为 `.env`：

```bash
cp ../.env.example ../.env
```

编辑 `.env` 文件，设置适合你本地环境的配置。

#### Step 4: 启动后端服务

```bash
cd backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

访问 `http://localhost:8000/docs` 查看 API 文档。

---

### 2.2 前端环境配置

#### Step 1: 安装依赖

```bash
cd frontend
npm install
```

#### Step 2: 启动开发服务器

```bash
npm run dev
```

访问 `http://localhost:5173` 查看前端页面。

## 3. Docker 环境配置

### 3.1 使用 Docker Compose 启动所有服务

#### Step 1: 确保在项目根目录

```bash
cd /path/to/FireTrain
```

#### Step 2: 启动服务

```bash
docker compose up -d
```

这会启动三个服务：
- `firetrain-mysql` (MySQL 数据库)
- `firetrain-backend` (FastAPI 后端)
- `firetrain-frontend` (Vue3 前端)

#### Step 3: 查看日志

```bash
docker compose logs -f
```

#### Step 4: 停止服务

```bash
docker compose down
```

#### Step 5: 清理数据（谨慎使用）

```bash
docker compose down -v  # 删除所有数据卷
```

## 4. 环境变量说明

### 4.1 核心环境变量

| 变量名 | 说明 | 默认值 | 示例 |
|--------|------|--------|------|
| `DATABASE_URL` | 数据库连接 URL | - | `sqlite:///./fire_training.db` |
| `JWT_SECRET_KEY` | JWT 加密密钥 | - | `your-secret-key` |
| `JWT_ALGORITHM` | JWT 算法 | `HS256` | `HS256` |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | Token 过期时间 | `30` | `30` |

### 4.2 服务端口

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `BACKEND_PORT` | 后端服务端口 | `8000` |
| `FRONTEND_PORT` | 前端服务端口 | `5173` |
| `VITE_API_BASE_URL` | 前端 API 地址 | `http://localhost:8000/api` |

### 4.3 AI 模型路径

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `MODEL_PATH` | 模型存储路径 | `./data/models` |
| `VIDEO_PATH` | 视频存储路径 | `./data/videos` |
| `YOLO_MODEL_NAME` | YOLO 模型名称 | `yolov8n.pt` |

## 5. 快速启动

### 5.1 方式一：本地开发模式（推荐）

```bash
# 终端 1: 启动后端
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload

# 终端 2: 启动前端
cd frontend
npm install
npm run dev
```

访问：
- 前端：http://localhost:5173
- 后端 API: http://localhost:8000
- API 文档：http://localhost:8000/docs

---

### 5.2 方式二：Docker 一键启动

```bash
docker compose up -d
```

访问：
- 前端：http://localhost:5173
- 后端 API: http://localhost:8000
- MySQL: localhost:3306

## 6. 常见问题

### Q1: 端口被占用怎么办？

修改 `.env` 文件中的端口配置：

```bash
BACKEND_PORT=8001  # 改为其他端口
FRONTEND_PORT=5174
```

### Q2: 前端无法连接后端？

检查以下几点：
1. 确认后端服务已启动
2. 检查 `VITE_API_BASE_URL` 配置是否正确
3. 检查 CORS 配置是否允许前端域名

### Q3: Docker 容器启动失败？

查看日志排查问题：

```bash
docker compose logs backend
docker compose logs frontend
```

### Q4: Python 依赖安装失败？

尝试以下方法：
1. 升级 pip: `pip install --upgrade pip`
2. 使用国内镜像：`pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple`
3. 检查 Python 版本是否符合要求

### Q5: npm install 失败？

尝试以下方法：
1. 清除缓存：`npm cache clean --force`
2. 删除 node_modules 重新安装：`rm -rf node_modules package-lock.json && npm install`
3. 使用国内镜像：`npm config set registry https://registry.npmmirror.com`
