## F3. API 请求参数和响应样例

### 3.1 用户注册接口

**请求**:
```bash
POST http://localhost:8000/api/user/register
Content-Type: application/json

{
  "username": "student001",
  "email": "student001@firetrain.com",
  "password": "Test123456",
  "phone": "13800138001"
}
```

**成功响应 (201)**:
```json
{
  "message": "注册成功",
  "user_id": 1
}
```

**失败响应 (400)**:
```json
{
  "detail": "用户名已存在",
  "code": "USER_EXISTS"
}
```

---

### 3.2 用户登录接口

**请求**:
```bash
POST http://localhost:8000/api/user/login
Content-Type: application/x-www-form-urlencoded

username=student001&password=Test123456
```

**成功响应 (200)**:
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VyX2lkIjoxLCJleHAiOjE3MTAzNDU2MDB9.abc123...",
```

## F5. 常见问题排查指南

### 问题 1: Token 鉴权失败

**现象**: 
```json
{
  "detail": "无法验证凭据",
  "code": 401
}
```

**排查步骤**:
1. 检查请求头是否包含 `Authorization: Bearer <token>`
2. 确认 Token 未过期（有效期 30 分钟）
3. 确认 Token 格式正确
4. 检查用户是否已被禁用
5. 查看后端日志确认 token 解码是否成功

**解决方案**:
```bash
# 重新登录获取新 token
curl -X POST http://localhost:8000/api/user/login \
  -d "username=student001&password=Test123456"
```

---

### 问题 2: 数据库连接失败

**现象**: 
```
sqlalchemy.exc.OperationalError: (sqlite3.OperationalError) unable to open database file
```

**排查步骤**:
1. 检查数据库文件路径是否正确
2. 确认数据库文件权限
3. 检查 Docker 卷挂载配置
4. 查看数据库连接字符串

**解决方案**:
```bash
# 检查数据库文件
ls -la backend/fire_training.db

# 修复权限
chmod 644 backend/fire_training.db

# 重启服务
make docker-restart
```

---

### 问题 3: 视频上传失败

**现象**: 
```json
{
  "detail": "视频上传失败",
  "code": "VIDEO_UPLOAD_FAILED"
}
```

**排查步骤**:
1. 检查文件大小是否超限（默认 100MB）
2. 确认上传目录有写权限
3. 检查文件格式是否支持（mp4, avi, mov）
4. 查看后端日志确认具体错误

**解决方案**:
```bash
# 检查上传目录权限
ls -la data/videos/

# 修复权限
chmod 755 data/videos/

# 检查文件大小
ls -lh your_video.mp4
```

---

### 问题 4: AI 评分超时

**现象**: 
```
Request timeout after 30000ms
```

**排查步骤**:
1. 确认 YOLOv8 模型文件是否存在
2. 检查模型加载是否成功
3. 查看 GPU/CPU 使用情况
4. 确认视频文件可访问

**解决方案**:
```bash
# 检查模型文件
ls -la backend/yolov8n.pt

# 查看服务日志
docker compose logs backend | grep -i "yolo\|model"

# 增加超时时间（前端配置）
VITE_REQUEST_TIMEOUT=60000
```

---

### 问题 5: 端口冲突

**现象**: 
```
OSError: [Errno 98] Address already in use
```

**排查步骤**:
1. 检查端口占用情况
2. 确认是否有重复服务运行
3. 查看进程列表

**解决方案**:
```bash
# 查看端口占用
lsof -i :8000
lsof -i :5173

# 释放端口
kill -SIGTERM <PID>

# 或者停止所有服务
make docker-down
make docker-up
```

---

### 问题 6: HTTPS 证书错误

**现象**: 
```
SSL: CERTIFICATE_VERIFY_FAILED
```

**排查步骤**:
1. 检查证书文件是否存在
2. 确认证书是否过期
3. 查看 HTTPS 配置

**解决方案**:
```bash
# 检查证书
ls -la certs/

# 生成自签名证书（开发环境）
openssl req -x509 -newkey rsa:4096 -keyout certs/key.pem -out certs/cert.pem -days 365 -nodes

# 重启服务
make docker-restart
```

---

### 问题 7: 跨域访问被拒绝

**现象**: 
```
Access to fetch at 'http://localhost:8000' from origin 'http://localhost:5173' has been blocked by CORS policy
```

**排查步骤**:
1. 检查后端 CORS 配置
2. 确认允许的来源地址
3. 查看浏览器控制台错误

**解决方案**:
```python
# backend/app/main.py 中配置
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5173"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```
