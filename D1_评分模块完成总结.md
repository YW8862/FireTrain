### 5. 数据结构设计

#### 评分结果 Schema
```python
{
    "total_score": Decimal,        # 总分 (0-100)
    "step_scores": List[StepScore], # 步骤分数列表
    "feedback": str,                # 总体反馈
    "suggestions": List[str],       # 改进建议列表
    "action_logs": List[ActionLog]  # 动作日志（可选）
}
```

#### StepScore Schema
```python
{
    "step_name": str,      # 步骤名称
    "score": Decimal,      # 该步骤得分 (0-100)
    "is_correct": bool,    # 是否正确
    "feedback": str,       # 步骤反馈
    "weight": float        # 权重 (0-1)
}
```

#### ActionLog Schema
```python
{
    "step_index": int,         # 步骤序号
    "step_name": str,          # 步骤名称
    "start_time": int,         # 开始时间（秒）
    "end_time": int,           # 结束时间（秒）
    "score": Decimal,          # 该步骤得分
    "is_correct": bool,        # 是否正确
    "description": str,        # 描述
    "key_points": List[str]    # 关键要点
}
```

## 技术实现要点

### 1. 评分规则设计

#### 步骤权重配置（灭火器操作）
```python
STEP_WEIGHTS = {
    "准备阶段": 0.2,      # 准备工作占 20%
    "提灭火器": 0.15,     # 提灭火器占 15%
    "拔保险销": 0.2,      # 拔保险销占 20%
    "握喷管": 0.1,        # 握喷管占 10%
    "瞄准火源": 0.2,      # 瞄准火源占 20%
    "压把手": 0.15,       # 压把手占 15%
}
```

#### 评分阈值
```python
SCORE_THRESHOLDS = {
    "excellent": 90,  # 优秀分数线
    "good": 80,       # 良好分数线
    "pass": 60,       # 合格分数线
}
```

#### 标准操作时间
```python
STANDARD_DURATION = {
    "fire_extinguisher": 120,  # 灭火器操作标准时间 2 分钟
}
```

### 2. 时间评分机制
- **合理时间范围**: 标准时间的 0.8-1.2 倍
  - 在此范围内：奖励 5 分
- **超时**: 超过标准时间的 1.5 倍
  - 扣除 5 分
- **最终分数限制**: 0-100 分之间

### 3. 难度系数映射
根据步骤的操作难度设置不同的基础分：
```python
difficulty_map = {
    "准备阶段": 0.9,      # 较简单
    "提灭火器": 0.85,     # 体力要求
    "拔保险销": 0.8,      # 技巧性操作
    "握喷管": 0.9,        # 相对简单
    "瞄准火源": 0.75,     # 关键步骤，难度较高
    "压把手": 0.8         # 需要持续用力
}
```

### 4. 反馈生成策略
根据分数段生成不同级别的反馈：
- **≥90 分**: "优秀！动作规范，流程熟练，继续保持"
- **80-89 分**: "良好！基本掌握操作要领，注意细节改进"
- **60-79 分**: "合格！但还有提升空间，建议多加练习"
- **<60 分**: "不合格！操作存在较大问题，请重新学习标准流程"

### 5. 改进建议生成
- 找出得分最低的 3 个步骤
- 根据分数段生成针对性建议：
  - <60 分：重点练习
  - 60-80 分：改进动作规范性
- ≥80 分时给出进阶建议
